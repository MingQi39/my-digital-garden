## 3. 后端架构设计

### 3.1 分层架构

```
┌─────────────────────────────────────┐
│            Handler 层                │  ← HTTP 入口，参数校验，响应序列化
│  职责：解析请求参数，调用 Service，    │     禁止包含任何业务逻辑
│       返回标准响应                    │
├─────────────────────────────────────┤
│          Application 层              │  ← 业务编排层
│  职责：事务管理，调用多个 Repository， │     Audit Log 在此写入
│       编排业务流程，写 Outbox         │     异步任务在此提交
├─────────────────────────────────────┤
│          Domain 层                   │  ← 领域模型（可选，复杂业务引入）
│  职责：核心业务规则、领域事件、         │     值对象、实体
│       不依赖任何基础设施               │
├─────────────────────────────────────┤
│         Repository 层                │  ← 数据访问层
│  职责：DB CRUD，Redis 操作            │     自动感知事务上下文
│       不包含业务逻辑                  │     返回领域模型而非 DB Model
├─────────────────────────────────────┤
│        Infrastructure 层             │  ← 基础设施
│  职责：DB/Redis/MQ/OSS 连接初始化     │     第三方 SDK 封装
│       配置加载，Wire 依赖注入          │
└─────────────────────────────────────┘
```

**层间调用规则：**

- Handler → Application（一对一调用）
- Application → Repository / Domain / Infrastructure（可调用多个）
- Repository → Infrastructure（通过依赖注入）
- **禁止反向依赖**
- **禁止 Handler 直接调用 Repository**

#### 3.2.4 Trace ID 全链路透传

**Middleware 实现：**

```go
func TraceMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        traceID := c.GetHeader("X-Trace-ID")
        if traceID == "" {
            traceID = uuid.NewString()
        }
        c.Set("trace_id", traceID)
        c.Writer.Header().Set("X-Trace-ID", traceID)

        ctx := context.WithValue(c.Request.Context(), ctxKeyTraceID{}, traceID)
        c.Request = c.Request.WithContext(ctx)

        c.Next()
    }
}

type ctxKeyTraceID struct{}

func TraceIDFromCtx(ctx context.Context) string {
    if v, ok := ctx.Value(ctxKeyTraceID{}).(string); ok {
        return v
    }
    return "unknown"
}
```

**透传规范：**

| 层级                 | 实施方式                                            |
| -------------------- | --------------------------------------------------- |
| 前端                 | 生成 `X-Trace-ID` Header                            |
| Gin Middleware       | 读取或生成，注入 context                            |
| Service / Repository | 从 `ctx` 获取 `trace_id`，日志中携带                |
| DB 查询              | GORM 通过 `WithContext(ctx)` 传递                   |
| Redis 操作           | 通过 `ctx` 传递                                     |
| 异步任务             | `trace_id` 写入 Task payload，Worker 恢复到 context |
| 跨服务调用           | HTTP Header / gRPC metadata 透传                    |

#### 3.3.3 写入接口

```go
type AuditRecord struct {
    TraceID      string
    UserID       int64
    UserName     string
    Action       string
    ResourceType string
    ResourceID   int64
    Before       interface{}
    After        interface{}
    IPAddr       string
    UserAgent    string
    Remark       string
}

type AuditLogger interface {
    Log(ctx context.Context, record AuditRecord) error
    LogBatch(ctx context.Context, records []AuditRecord) error
}

type auditLogger struct {
    db *gorm.DB
}

func (a *auditLogger) Log(ctx context.Context, record AuditRecord) error {
    if record.TraceID == "" {
        record.TraceID = TraceIDFromCtx(ctx)
    }

    db := GetDB(ctx, a.db)

    beforeJSON, _ := json.Marshal(record.Before)
    afterJSON, _ := json.Marshal(record.After)

    return db.Exec(`
        INSERT INTO audit_logs (trace_id, user_id, user_name, action,
            resource_type, resource_id, before_data, after_data, ip_addr, user_agent, remark)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)`,
        record.TraceID, record.UserID, record.UserName, record.Action,
        record.ResourceType, record.ResourceID, beforeJSON, afterJSON,
        record.IPAddr, record.UserAgent, record.Remark,
    ).Error
}
```

#### 3.3.4 写入规范

| 规则         | 说明                                                                       |
| ------------ | -------------------------------------------------------------------------- |
| 写入时机     | 仅在 Application 层写入                                                    |
| 事务归属     | 审计日志 **必须与业务操作在同一事务内**                                    |
| 禁止项       | 禁止 Repository 自动写审计日志                                             |
| 必填字段     | `action`, `resource_type`, `resource_id`, `user_id`                        |
| Before/After | UPDATE 必须同时记录变更前后数据；CREATE 仅记录 After；DELETE 仅记录 Before |

### 3.4 事务管理

#### 3.4.1 Context Key 定义（类型安全）

```go
type ctxKey struct{ name string }

var ctxKeyTx = ctxKey{"tx"}

func CtxWithTx(ctx context.Context, tx *gorm.DB) context.Context {
    return context.WithValue(ctx, ctxKeyTx, tx)
}

func GetDB(ctx context.Context, fallback *gorm.DB) *gorm.DB {
    if tx, ok := ctx.Value(ctxKeyTx).(*gorm.DB); ok {
        return tx
    }
    return fallback.WithContext(ctx)
}
```

#### 3.4.2 事务管理器

```go
type TxManager struct {
    db     *gorm.DB
    logger *zap.Logger
}

func NewTxManager(db *gorm.DB, logger *zap.Logger) *TxManager {
    return &TxManager{db: db, logger: logger}
}

func (t *TxManager) WithTx(ctx context.Context, fn func(ctx context.Context) error) error {
    traceID := TraceIDFromCtx(ctx)

    return t.db.WithContext(ctx).Transaction(func(tx *gorm.DB) error {
        txCtx := CtxWithTx(ctx, tx)

        if err := fn(txCtx); err != nil {
            t.logger.Warn("transaction rolled back",
                zap.String("trace_id", traceID),
                zap.Error(err),
            )
            return err
        }
        return nil
    })
}

// ReadOnly 提供只读事务（SELECT 场景，防止意外写入）
func (t *TxManager) ReadOnly(ctx context.Context, fn func(ctx context.Context) error) error {
    return t.db.WithContext(ctx).Transaction(func(tx *gorm.DB) error {
        tx = tx.Exec("SET TRANSACTION READ ONLY")
        txCtx := CtxWithTx(ctx, tx)
        return fn(txCtx)
    })
}
```

#### 3.4.3 Repository 自动感知事务

```go
type SampleRepository struct {
    db *gorm.DB
}

func (r *SampleRepository) Create(ctx context.Context, sample *Sample) error {
    return GetDB(ctx, r.db).Create(sample).Error
}

func (r *SampleRepository) Update(ctx context.Context, sample *Sample) error {
    return GetDB(ctx, r.db).Save(sample).Error
}

func (r *SampleRepository) GetByID(ctx context.Context, id int64) (*Sample, error) {
    var sample Sample
    err := GetDB(ctx, r.db).First(&sample, id).Error
    return &sample, err
}
```

#### 3.4.4 事务使用规范

| 规则            | 说明                                               |
| --------------- | -------------------------------------------------- |
| 事务开启位置    | **只在 Application 层开启**                        |
| Repository 行为 | 自动感知 context 中的 tx，无需显式传参             |
| 禁止嵌套        | 禁止在 WithTx 内部再调用 WithTx                    |
| 禁止跨服务      | 禁止跨服务事务，使用 Outbox Pattern 保证最终一致性 |
| 超时控制        | 事务必须设置超时 `context.WithTimeout`，默认 30s   |
| 读操作          | 纯读操作使用 `ReadOnly` 事务或直接查询             |

#### 3.4.5 Outbox Pattern（异步一致性）

```go
func (s *SampleService) CreateWithNotify(ctx context.Context, req CreateSampleReq) error {
    return s.txMgr.WithTx(ctx, func(ctx context.Context) error {
        sample := buildSample(req)
        if err := s.sampleRepo.Create(ctx, sample); err != nil {
            return err
        }

        // 在同一事务内写 Outbox
        return s.outboxRepo.Create(ctx, &OutboxMessage{
            AggregateType: "sample",
            AggregateID:   sample.ID,
            EventType:     "sample.created",
            Payload:       toJSON(sample),
        })
    })
    // Outbox Poller 异步读取并投递到消息队列
}
```

```go
func GetSampleWithCache(ctx context.Context, id int64) (*Sample, error) {

    if cached, err := redis.Get(ctx, key); err == nil {

        json.Unmarshal([]byte(cached), &s)
        return &s, nil

    // singleflight 防击穿：同一 key 的并发请求只查一次 DB
    v, err, _ := sfGroup.Do(key, func() (interface{}, error) {
        sample, err := sampleRepo.GetByID(ctx, id)

            return nil, err

        // TTL 随机化防雪崩：基础 10min + 随机 0~5min
        ttl := 10*time.Minute + time.Duration(rand.Intn(300))*time.Second
        data, _ := json.Marshal(sample)
        redis.Set(ctx, key, data, ttl)
        return sample, nil
    })

    if err != nil {
        return nil, err
    }
    return v.(*Sample), nil
```

### 3.8 幂等性设计

#### 3.8.1 设计方案

```go
func IdempotencyMiddleware(redis *redis.Client) gin.HandlerFunc {
    return func(c *gin.Context) {
        if c.Request.Method == "GET" || c.Request.Method == "HEAD" {
            c.Next()
            return
        }

        key := c.GetHeader("X-Idempotency-Key")
        if key == "" {
            c.Next()
            return
        }

        cacheKey := fmt.Sprintf("lims:idempotency:%s", key)

        // 尝试获取已缓存的响应
        if cached, err := redis.Get(c.Request.Context(), cacheKey).Bytes(); err == nil {
            var resp Response
            json.Unmarshal(cached, &resp)
            c.AbortWithStatusJSON(200, resp)
            return
        }

        // 尝试获取锁（防止并发重复提交）
        locked, err := redis.SetNX(c.Request.Context(), cacheKey+":lock", "1", 30*time.Second).Result()
        if err != nil || !locked {
            c.AbortWithStatusJSON(409, Response{Code: 40900, Message: "请求正在处理中，请勿重复提交"})
            return
        }
        defer redis.Del(c.Request.Context(), cacheKey+":lock")

        // 包装 ResponseWriter 以捕获响应
        w := &responseCapture{ResponseWriter: c.Writer}
        c.Writer = w

        c.Next()

        // 成功响应缓存 24 小时
        if c.Writer.Status() >= 200 && c.Writer.Status() < 300 {
            redis.Set(c.Request.Context(), cacheKey, w.body.Bytes(), 24*time.Hour)
        }
    }
}
```

```go
    return func(c *gin.Context) {
        userID, _ := c.Get("user_id")
        key := fmt.Sprintf("lims:ratelimit:%v:%s", userID, c.FullPath())
```

```go
    Roles    []string `json:"roles"`

func GenerateTokenPair(user *User) (accessToken, refreshToken string, err error) {
    // Access Token: 短有效期
    accessClaims := Claims{
        TenantID: user.TenantID,
        UserID:   user.ID,
        UserName: user.Name,
        Roles:    user.Roles,
        RegisteredClaims: jwt.RegisteredClaims{
            ExpiresAt: jwt.NewNumericDate(time.Now().Add(2 * time.Hour)),
            IssuedAt:  jwt.NewNumericDate(time.Now()),
            Issuer:    "lims",
        },
    }
    accessToken, err = jwt.NewWithClaims(jwt.SigningMethodHS256, accessClaims).
        SignedString([]byte(config.JWTSecret))
    if err != nil {
        return
    }

    // Refresh Token: 长有效期
    refreshClaims := jwt.RegisteredClaims{
        ExpiresAt: jwt.NewNumericDate(time.Now().Add(7 * 24 * time.Hour)),
        IssuedAt:  jwt.NewNumericDate(time.Now()),
        Subject:   fmt.Sprintf("%d", user.ID),
        Issuer:    "lims",
    }
    refreshToken, err = jwt.NewWithClaims(jwt.SigningMethodHS256, refreshClaims).
        SignedString([]byte(config.RefreshSecret))
    return
}
```

```go
func HashPassword(password string) (string, error) {

// 密码强度校验
func ValidatePassword(password string) error {
    if len(password) < 8 {
        return errors.New("密码长度至少 8 位")

    var hasUpper, hasLower, hasDigit bool
    for _, c := range password {
        switch {
        case unicode.IsUpper(c): hasUpper = true
        case unicode.IsLower(c): hasLower = true
        case unicode.IsDigit(c): hasDigit = true
        }
    }
    if !hasUpper || !hasLower || !hasDigit {
        return errors.New("密码必须包含大写字母、小写字母和数字")
    }
    return nil
}
```

```go
    Download(ctx context.Context, key string) (io.ReadCloser, error)
    Delete(ctx context.Context, key string) error
    GetURL(ctx context.Context, key string, expiry time.Duration) (string, error)
}

func (s *FileService) Upload(ctx context.Context, file multipart.File, header *multipart.FileHeader) (*FileRecord, error) {
    // 文件大小校验（最大 100MB）
    if header.Size > 100*1024*1024 {
        return nil, WrapError(ErrBadRequest, "文件大小不能超过 100MB")
    }

    // 文件类型白名单
    ext := strings.ToLower(filepath.Ext(header.Filename))
    if !isAllowedExt(ext) {
        return nil, WrapError(ErrBadRequest, "不支持的文件类型")
    }

    // 生成存储路径：年/月/日/uuid.ext
    key := fmt.Sprintf("%s/%s%s", time.Now().Format("2006/01/02"), uuid.NewString(), ext)

    if err := s.storage.Upload(ctx, key, file, header.Size); err != nil {
        return nil, err
    }

    record := &FileRecord{
        FileName:    header.Filename,
        StorageKey:  key,
        FileSize:    header.Size,
        ContentType: header.Header.Get("Content-Type"),
        UploadedBy:  CurrentUserID(ctx),
    }
    if err := s.db.WithContext(ctx).Create(record).Error; err != nil {
        return nil, err
    }
    return record, nil
}
```

```go
    sig := <-quit
    logger.Info("shutting down", zap.String("signal", sig.String()))

    // 给 30s 处理剩余请求
    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()

    // 按序关闭
    // 1. HTTP Server（停止接受新请求，等待活跃请求完成）
    if err := srv.Shutdown(ctx); err != nil {
        logger.Error("server shutdown error", zap.Error(err))

    // 2. Asynq Worker（停止拉取新任务，等待当前任务完成）
    asynqServer.Shutdown()
    // 3. 关闭 DB 连接池
    sqlDB, _ := db.DB()
    sqlDB.Close()
    // 4. 关闭 Redis
    redisClient.Close()

    logger.Info("server exited gracefully")
```

```typescript
// api/client.ts
import axios, { AxiosError, InternalAxiosRequestConfig } from "axios";
import { v4 as uuidv4 } from "uuid";
import { useAuthStore } from "@/stores/authStore";

const client = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 30000,
});

// 请求拦截器
client.interceptors.request.use((config: InternalAxiosRequestConfig) => {
  // 注入 Trace ID
  config.headers["X-Trace-ID"] = uuidv4();

  // 注入 Token
  const token = useAuthStore.getState().accessToken;
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }

  // 写操作注入幂等键
  if (["post", "put"].includes(config.method ?? "")) {
    config.headers["X-Idempotency-Key"] = config.headers["X-Idempotency-Key"] || uuidv4();
  }

  return config;
});

// 响应拦截器
client.interceptors.response.use(
  (res) => {
    const data = res.data;
    if (data.code !== 0) {
      return Promise.reject(new BizError(data.code, data.message, data.trace_id));
    }
    return data.data;
  },
  async (error: AxiosError<ApiResponse>) => {
    const status = error.response?.status;
    const data = error.response?.data;

    // 401: Token 过期，尝试刷新
    if (status === 401) {
      const refreshed = await tryRefreshToken();
      if (refreshed && error.config) {
        return client.request(error.config);
      }
      useAuthStore.getState().logout();
      window.location.href = "/login";
      return Promise.reject(error);
    }

    // 403: 无权限
    if (status === 403) {
      message.error("没有操作权限");
      return Promise.reject(error);
    }

    // 429: 限流
    if (status === 429) {
      message.warning("操作过于频繁，请稍后再试");
      return Promise.reject(error);
    }

    // 其他错误
    message.error(data?.message || "网络错误，请稍后重试");
    return Promise.reject(error);
  },
);

class BizError extends Error {
  code: number;
  traceId: string;
  constructor(code: number, message: string, traceId: string) {
    super(message);
    this.code = code;
    this.traceId = traceId;
  }
}

async function tryRefreshToken(): Promise<boolean> {
  const refreshToken = useAuthStore.getState().refreshToken;
  if (!refreshToken) return false;
  try {
    const res = await axios.post("/api/v1/auth/refresh", null, {
      headers: { "X-Refresh-Token": refreshToken },
    });
    useAuthStore.getState().setTokens(res.data.data.access_token, res.data.data.refresh_token);
    return true;
  } catch {
    return false;
  }
}

export default client;
```

## 5. 数据库设计与运维

### 5.0 共享 PostgreSQL：多端同一数据源（开发 / 联调）

**托管路线（与第 2 章一致）：** **现阶段**使用 [Supabase](https://supabase.com/) 托管 PostgreSQL（开发 / 测试 / 早期生产可按环境拆 Project 或共用）；**后续**迁移至 **阿里云 RDS PostgreSQL**，与阿里云 VPC、ECS/ACK、备份与合规策略统一。二者均为标准 PostgreSQL，业务层保持与厂商无关的 SQL / GORM 写法，切换以 **连接串、网络访问控制、连接池上限** 为主。

**要解决什么问题：** 若每人本机各跑一套 Postgres，或应用连 A 库、DBeaver 连 B 库，则「A 机器插入一行，B 机器看不到」是预期现象——因为根本不是同一个数据库。**解决办法**是：在团队都能访问的**同一托管实例**上，让所有环境使用**同一套连接信息**：`Host`（或连接串里的 host）、`Port`、`Database`（库名）、`User`、`Password`。事务提交后，任意客户端（本机 API、同事电脑上的前端+后端、DBeaver、脚本）连上去看到的都是同一份数据。

**分阶段落地形态：**

| 阶段 | 形态 | 说明 |
| --- | --- | --- |
| **当前** | Supabase | 在 Project → **Settings → Database** 获取连接串；区分 **直连** 与 **Pooler**（端口常为 `6543`）：Serverless/边缘场景多用 Pooler，注意与 GORM **prepared statement**、会话模式的兼容性（按官方文档选用 Session / Transaction 模式）。控制台可配置 **Network restrictions**、始终 **SSL**。关注套餐的 **最大连接数**，应用 `MaxOpenConns` 需低于上限并预留 DBeaver / 脚本额度。 |
| **后续** | 阿里云 RDS PostgreSQL | 与 ACK/ECS **同地域同 VPC** 走**内网地址**；**白名单**仅允许办公网 / VPN / 安全组。可按需在应用与 RDS 之间加 **PgBouncer**（与第 6.4 节一致），只读分析可走**只读实例**地址。 |
| 可选 | 自建 VM / 其他云 RDS | 内网可达、统一发连接串即可；原则与下文安全 checklist 相同。 |

**Supabase → 阿里云 RDS 迁移（概要）：**

1. **选型：** 小到中等库常用 **`pg_dump` / `pg_restore`**（逻辑备份）；要求更少停机时再评估逻辑复制等方案（按数据量与窗口与运维能力定）。  
2. **顺序：** 停写或只读窗口 → 全量备份与恢复到 RDS → 在 RDS 上跑 migration 校验 → 更新 K8s **Secret** 中的 `DATABASE_URL`（或分字段配置）→ **滚动重启** API / Worker。  
3. **并行验证：** 应用指 RDS、回归核心链路后再下线 Supabase 上的写流量。  
4. **迁完后：** 按阿里云侧白名单、SSL、账号拆分收紧权限；连接数充裕时再上 PgBouncer。

**与本文架构的关系：** Supabase 阶段多为**直连或官方 Pooler**，应用需**主动控制连接池大小**；迁至 **RDS** 后，生产仍推荐经 **PgBouncer**、VPC 与最小权限账号接入（与全文 PgBouncer 章节一致）。无论哪一阶段，**必须**把下面的安全项当作上线 checklist，而不是可选项。

**网络与访问控制（必做）：**

- **安全组 / 云防火墙 / 本机 `iptables`：** 对 `5432`（或 PgBouncer 的 `6432`）**不要**对 `0.0.0.0/0` 裸放；改为仅允许 **办公室出口 IP**、**跳板机** 或 **VPN 网段**（如 `10.8.0.0/24`）。
- **优先内网：** 应用与数据库同 VPC、同专线或 VPN，公网暴露数据库端口是最后手段；若必须公网，至少 **IP 白名单 + 强认证 + TLS**。
- **TLS：** 客户端设置 `sslmode=require`（或云厂商要求的 `verify-full` / `verify-ca`），避免口令与数据在链路上明文传输。
- **是否只允许 VPN：** 若团队已统一零信任 / 办公 VPN，可将数据库仅开在 VPN 内网地址上，从根上缩小攻击面。

**账号与口令：**

- 使用**足够强度**的数据库口令，存放在 **Secret / 密钥管理** 或团队私密渠道，**不要**写进仓库。
- **应用账号**与 **只读分析账号**（如 DBeaver 日常查询）分离；生产与开发库账号权限分离，遵循最小权限。
- 定期轮换口令；离职回收数据库账号。

**客户端对齐：**

- 后端 `DATABASE_URL` / 分字段配置与 DBeaver 的 Host、Port、Database、User、Password（及 SSL 选项）**指向同一实例、同一库**；端口误填、连到 `localhost`、或 SSL 模式不一致会导致「以为连上了共享库实则没有」的误判。

**协作注意：** 多人共用一套开发库时，迁移（Migration）与破坏性脚本需要约定执行人；必要时为个人实验使用**独立 schema** 或独立 `database`，避免互相踩数据。

```go
type gormLogger struct {

func (l *gormLogger) Trace(ctx context.Context, begin time.Time, fc func() (sql string, rowsAffected int64), err error) {
    elapsed := time.Since(begin)
    sql, rows := fc()
    traceID := TraceIDFromCtx(ctx)

    switch {
    case err != nil && !errors.Is(err, gorm.ErrRecordNotFound):
        l.logger.Error("db error",
            zap.String("trace_id", traceID),
            zap.Error(err),
            zap.String("sql", sql),
            zap.Duration("elapsed", elapsed),
        )
    case elapsed > l.slowThreshold:
        l.logger.Warn("slow query",
            zap.String("trace_id", traceID),
            zap.String("sql", sql),
            zap.Int64("rows", rows),
            zap.Duration("elapsed", elapsed),
        )
    }
}
```

## 6. 数据扩展性设计（ToB 大数据量）

系统面向 ToB 场景，需支撑多企业客户的大规模数据量。本章涵盖多租户隔离、业务表分区、读写分离、连接池管理、索引策略、全文搜索及业务数据归档。

### 6.1 多租户隔离架构

#### 6.1.1 隔离模式选择

采用**混合隔离模式**，按客户规模和合规要求分层：


| 隔离模式                | 适用客户          | 数据隔离级别    | 运维成本 | 扩展性             |
| ------------------- | ------------- | --------- | ---- | --------------- |
| 共享数据库 + `tenant_id` | 中小客户（占 80%+）  | 逻辑隔离      | 低    | 高（一套部署服务所有客户）   |
| 独立 Schema           | 中型客户 / 行业合规要求 | Schema 隔离 | 中    | 中（Schema 数量有上限） |
| 独立数据库               | 大客户 / 私有化部署   | 物理隔离      | 高    | 按需独立扩展          |


默认采用"共享数据库 + `tenant_id`"模式，所有业务表均包含 `tenant_id` 字段。

#### 6.1.3 Middleware 自动注入 tenant_id

```go
func TenantMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        claims := GetClaims(c.Request.Context())
        if claims == nil {
            Fail(c, ErrUnauthorized)
            c.Abort()
            return
        }

        tenantID := claims.TenantID
        if tenantID == 0 {
            Fail(c, WrapError(ErrForbidden, "未关联租户"))
            c.Abort()
            return
        }

        ctx := context.WithValue(c.Request.Context(), ctxKeyTenantID{}, tenantID)
        c.Request = c.Request.WithContext(ctx)
        c.Next()
    }
}

type ctxKeyTenantID struct{}

func TenantIDFromCtx(ctx context.Context) int64 {
    if v, ok := ctx.Value(ctxKeyTenantID{}).(int64); ok {
        return v
    }
    return 0
}
```

```go
import "gorm.io/plugin/dbresolver"

func InitDBWithReplicas(cfg DatabaseConfig) (*gorm.DB, error) {
    db, err := gorm.Open(postgres.Open(cfg.PrimaryDSN), &gorm.Config{
        SkipDefaultTransaction: true,
    })
    if err != nil {
        return nil, err
    }

    var replicaDialectors []gorm.Dialector
    for _, dsn := range cfg.ReplicaDSNs {
        replicaDialectors = append(replicaDialectors, postgres.Open(dsn))
    }

    err = db.Use(dbresolver.Register(dbresolver.Config{
        Replicas: replicaDialectors,
        Policy:   dbresolver.RandomPolicy{},
    }).
        SetMaxOpenConns(cfg.MaxOpenConns).
        SetMaxIdleConns(cfg.MaxIdleConns).
        SetConnMaxLifetime(30 * time.Minute).
        SetConnMaxIdleTime(5 * time.Minute))

    return db, err
}
```

### 6.5 索引策略

#### 6.5.1 通用索引规则


| 索引类型        | 适用场景       | 注意事项                |
| ----------- | ---------- | ------------------- |
| B-Tree 联合索引 | 等值 + 范围查询  | 高选择性列在前，范围列在后       |
| GIN 索引      | JSONB 字段查询 | 写入开销较大，仅对高频查询字段建立   |
| 部分索引        | 特定状态过滤     | 减少索引体积，提升写入性能       |
| 覆盖索引        | 避免回表       | 用 INCLUDE 子句包含常用查询列 |

### 6.6 全文搜索

#### 6.6.1 初期方案：PostgreSQL 内置全文搜索

```sql
-- 给样品表添加搜索向量列
ALTER TABLE samples ADD COLUMN search_vector tsvector;

-- 自动更新搜索向量
CREATE OR REPLACE FUNCTION samples_search_trigger() RETURNS trigger AS $$
BEGIN
    NEW.search_vector :=
        setweight(to_tsvector('simple', coalesce(NEW.sample_no, '')), 'A') ||
        setweight(to_tsvector('simple', coalesce(NEW.name, '')), 'A') ||
        setweight(to_tsvector('simple', coalesce(NEW.source, '')), 'B');
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_samples_search
    BEFORE INSERT OR UPDATE ON samples
    FOR EACH ROW EXECUTE FUNCTION samples_search_trigger();

-- GIN 索引
CREATE INDEX idx_samples_search ON samples USING GIN (search_vector);

-- 搜索查询
SELECT * FROM samples
WHERE tenant_id = 1001
  AND search_vector @@ plainto_tsquery('simple', '血液 常规')
ORDER BY ts_rank(search_vector, plainto_tsquery('simple', '血液 常规')) DESC;
```

| 数据状态 | 时间范围  | 存储位置           | 查询方式 | 操作    |
| ---- | ----- | -------------- | ---- | ----- |
| 活跃数据 | 近 1 年 | 主库热分区          | 在线查询 | 正常读写  |
| 历史数据 | 1-3 年 | 冷分区 / 只读副本     | 按需查询 | 只读    |
| 归档数据 | 3 年+  | 对象存储（S3/MinIO） | 导出下载 | 导入后查看 |


#### 6.7.2 归档 Worker 实现

```go
func HandleArchivePartition(ctx context.Context, t *asynq.Task) error {
    var payload ArchivePayload
    if err := json.Unmarshal(t.Payload(), &payload); err != nil {
        return err
    }

    partitionName := payload.PartitionName // e.g. "samples_2023_q1"

    // 1. 导出分区数据到 CSV
    csvPath, err := exportPartitionToCSV(ctx, partitionName)
    if err != nil {
        return fmt.Errorf("export partition: %w", err)
    }

    // 2. 上传到对象存储
    objectKey := fmt.Sprintf("archive/%s/%s.csv.gz", time.Now().Format("2006"), partitionName)
    if err := storage.Upload(ctx, objectKey, csvPath); err != nil {
        return fmt.Errorf("upload to storage: %w", err)
    }

    // 3. 记录归档日志
    if err := recordArchiveLog(ctx, partitionName, objectKey); err != nil {
        return fmt.Errorf("record archive log: %w", err)
    }

    // 4. DETACH 分区（数据仍在，但不再参与查询）
    if err := detachPartition(ctx, payload.ParentTable, partitionName); err != nil {
        return fmt.Errorf("detach partition: %w", err)
    }

    logger.Info("partition archived",
        zap.String("partition", partitionName),
        zap.String("object_key", objectKey),
    )
    return nil
}
```

#### 6.7.3 归档定时调度

```go
// 每月 1 日凌晨 3 点执行归档检查
scheduler.Register("@monthly", asynq.NewTask("archive:check", nil))

func HandleArchiveCheck(ctx context.Context, t *asynq.Task) error {
    tables := []string{"samples", "experiments", "experiment_results", "reports"}
    archiveThreshold := time.Now().AddDate(-3, 0, 0) // 3 年前

    for _, table := range tables {
        partitions, err := getPartitionsOlderThan(ctx, table, archiveThreshold)
        if err != nil {
            return err
        }
        for _, p := range partitions {
            _, err := asynqClient.Enqueue(asynq.NewTask("archive:partition", toJSON(ArchivePayload{
                ParentTable:   table,
                PartitionName: p,
            })))
            if err != nil {
                logger.Error("enqueue archive task failed", zap.String("partition", p), zap.Error(err))
            }
        }
    }
    return nil
}
```

### 6.8 数据量估算与容量规划

以 100 个企业客户、运营 3 年为基准：


| 表    | 单客户年增长      | 100 客户 3 年总量 | 单行大小（估算）     | 总存储         |
| ---- | ----------- | ------------ | ------------ | ----------- |
| 样品   | 5 万-50 万行   | 1.5 亿-15 亿   | ~500B        | 75GB-750GB  |
| 实验记录 | 10 万-100 万行 | 3 亿-30 亿     | ~1KB         | 300GB-3TB   |
| 实验结果 | 20 万-200 万行 | 6 亿-60 亿     | ~2KB (JSONB) | 1.2TB-12TB  |
| 审计日志 | 50 万-500 万行 | 15 亿-150 亿   | ~1KB         | 1.5TB-15TB  |
| 报告   | 1 万-10 万行   | 3000 万-3 亿   | ~5KB         | 150GB-1.5TB |


**容量规划建议：**

- 主库 SSD 存储：起步 1TB，按年扩容
- 只读副本：与主库同等存储
- 对象存储：用于归档数据，按需扩容（成本约为 SSD 的 1/10）
- 配合分区 + 归档策略，主库在线数据控制在 1-2TB 以内

## 10. 全链路流程

### 10.1 同步请求链路

```
用户操作（点击按钮）
    │
    ▼
[前端] 生成 trace_id → Axios 请求拦截器
    │ 携带: Authorization / X-Trace-ID / X-Idempotency-Key
    ▼
[Ingress] TLS 终止 → Rate Limit 检查 → 转发
    │
    ▼
[Gin Middleware 链]
    │ Recovery → Logger → TraceID → Auth → RBAC → RateLimit → Idempotency → Validator
    ▼
[Handler 层]
    │ 参数校验（binding + validator）
    │ 调用 Service，传入 ctx
    ▼
[Application / Service 层]
    │ 开启事务 (WithTx)
    │  ├─ Repository.GetByID → 获取旧数据
    │  ├─ 业务逻辑处理
    │  ├─ Repository.Update → 写 DB
    │  ├─ AuditLog.Log → 写审计日志（同一事务）
    │  └─ Outbox.Create → 写 Outbox 消息（如需异步通知）
    │ 提交事务
    │ 清除缓存
    ▼
[Handler 层] 返回标准 Response
    │
    ▼
[Gin Middleware] 记录 Access Log（带 trace_id + latency）
    │
    ▼
[前端] Axios 响应拦截器 → 业务判断 → React Query 缓存更新 → UI 渲染
```

### 10.2 异步任务链路

```
[用户操作] 点击"导出报告"
    │
    ▼
[API Handler] 接收请求
    │
    ▼
[Service 层]
    │ WithTx:
    │  ├─ 创建 AsyncTask 记录 (status=PENDING)
    │  └─ 提交事务
    │ 事务成功后: asynqClient.Enqueue(task)
    │ 返回 task_id
    ▼
[前端] 收到 task_id → 开始轮询 useTaskPolling(taskId)
    │
    │         ┌──────────────────────────────┐
    │         │       Asynq / Redis 队列      │
    │         └──────────────┬───────────────┘
    │                        │ Worker 拉取
    │                        ▼
    │         ┌──────────────────────────────┐
    │         │       Worker Pod              │
    │         │  1. 恢复 trace_id             │
    │         │  2. updateStatus(PROCESSING)  │
    │         │  3. 执行业务逻辑              │
    │         │  4. updateStatus(SUCCESS/FAIL)│
    │         └──────────────────────────────┘
    │
    ▼
[前端轮询] GET /api/v1/tasks/{id}
    │ status=PROCESSING → 显示进度
    │ status=SUCCESS → 显示下载链接，停止轮询
    │ status=FAILED → 显示错误，停止轮询
```

```
                              │                         │
```
